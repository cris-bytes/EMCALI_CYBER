# Informes Técnicos y Documentación Oficial

Esta carpeta contiene toda la documentación extensa y formal del proyecto, requerida para auditorías, transferencias de conocimiento y aprobaciones gerenciales.

## Contenido

- **`EMCALI_Cyber20_Arquitectura_Capas.pdf`** (y `.docx`): 
  - Define el patrón arquitectónico utilizado (Capas).
  - Explica la separación entre Presentación (Streamlit), Lógica de Negocio (Core) y Acceso a Datos.
  - Útil para arquitectos de software que deseen integrar esta app con otros sistemas del SOC.

- **`EMCALY_Cyber_2_0 (3).pdf`**: 
  - Presentación general del ecosistema Cyber 2.0.
  - Contexto de ciberseguridad, alcance y limitaciones de la herramienta.

- **`Informe Técnico 5.pdf`** (y `.docx`): 
  - Reporte iterativo sobre el progreso del desarrollo de módulos de IA.
  - Documenta pruebas de rendimiento de los LLMs.

- **`Informe_Tecnico_EMCALI_Cyber20_Hibrido_v3.pdf`** (y `.docx`): 
  - Documento core. Explica exhaustivamente la naturaleza "Híbrida" (uso simultáneo de IA en local para datos sensibles e IA en nube para datos genéricos).
  - Políticas de privacidad de datos, y resultados de evaluaciones de precisión en remediaciones.

- **`Informe_Tecnico_EMCALI_Cyber20_Hibrido_v4_Ollama.docx`**: 
  - Evolución del informe V3, enfocándose fuertemente en el despliegue del componente local con **Ollama**.
  - Documenta el hardware requerido y benchmarks de tiempos de inferencia utilizando modelos LLama 3 / Mistral en local.
